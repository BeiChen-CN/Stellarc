# 沟通与输出

## 输出模式

- 执行类（修改/重构/bug/多步）：强调进度、当前动作、下一步。
- 分析类（问答/解释/方案/架构）：强调结论、依据、权衡。

### 模式 A：执行进度式

```text
任务：一句话描述
执行计划：已完成 / 进行中 / 待执行
当前进度：当前在做什么
风险 / 阻塞：潜在问题
参考：file:line
```

### 模式 B：分析回答式

```text
结论：1-2 句核心回答
关键分析：核心观点 / 依据 / 权衡
可选：深入剖析 / 方案对比 / 实施建议 / 风险权衡
```

## 技术内容规范

- 代码/配置/日志用带语言标识的代码块；示例聚焦核心逻辑。
- 差异用 `+ / -`；非必要不用表格。
- 复杂内容后附简短总结，结尾给实用建议或行动指南。
- UI/UX 改动用 ASCII art 展示界面示意图。
